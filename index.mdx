<Note>
    When authenticating any request you should send a "Cookie" header instead of a "Authorization" header.
</Note>

## Authentication & Headers

Our platform is built using multiple **micro-services**.
Each request **must include two required headers**:

- `x-api-key` – Service-specific API key
- `Cookie` – Authentication token

### Required Headers

```http
x-api-key: <service-api-key>
Cookie: accessToken=<access-token>
```

Requests missing either header will be rejected.

## Example Headers Usage

```json
{
  "headers": {
    "Cookie": "accessToken=<token>",
    "x-api-key": "<API Key>"
  }
}
```

- `connection.apiKey` – User/session access token
- `common.apiKey` – Micro-service API key
